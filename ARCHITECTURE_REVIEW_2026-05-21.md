# CMS Lite 完善建议（审计日期：2026-05-21）

## P0（优先修复）

1. **密码哈希方案需要升级**
   - 当前使用纯 SHA-256 无盐哈希，抗暴力破解能力不足。
   - 建议迁移到 `argon2id` 或 `bcrypt`（含随机盐、成本因子），并加入登录失败限速。

2. **JWT 默认密钥存在高风险默认值**
   - 当前 `JWT_SECRET` 缺失时会回退到硬编码默认值，生产环境可能被误用。
   - 建议在生产环境强制校验：未配置即拒绝启动/拒绝签发。

3. **Admin 仅校验 Cookie 是否存在，未在中间件验签**
   - 中间件只检查 `cms_token` 是否存在，没有验证有效性/过期。
   - 建议在中间件校验 JWT（至少校验签名与过期），失败即跳转登录。

4. **登录接口缺少暴力破解防护**
   - 目前没有 IP/账号维度限流和锁定策略。
   - 建议加 Cloudflare WAF/Turnstile、应用层速率限制（KV + sliding window）。

## P1（建议近期完成）

1. **权限模型需要更细粒度 RBAC**
   - 当前 `requireAdmin` 将 `admin/editor` 合并授权，且接口层分散校验。
   - 建议按资源与动作定义权限矩阵（content:read/write/publish, user:manage 等）。

2. **可观测性不足（审计日志/告警）**
   - 缺少关键操作审计（登录、权限变更、发布、删除、API Key 操作）。
   - 建议新增 `audit_logs` 表 + 管理台查询 + 异常告警。

3. **安全 Cookie 策略可进一步增强**
   - 建议补充 `Domain` 策略（多站点时）、以及可配置 `SameSite=Strict` 选项。
   - 可增加短 Access Token + Refresh Token 轮换机制。

4. **API 防滥用策略**
   - 已有 API Key 能力，但建议加配额、过期、按路由限流、最近使用 IP 画像。

## P2（中期优化）

1. **搜索能力升级**
   - 当前主要是 SQLite FTS，建议加入拼写容错、同义词、权重排序、热门查询缓存。

2. **内容发布工作流**
   - 增加草稿评审、多人协作评论、定时发布冲突检测、回滚与差异对比。

3. **插件/主题沙箱治理**
   - 对插件权限、生命周期钩子、版本兼容矩阵做统一规范。

4. **备份与灾备**
   - 增加 D1 定时导出 + R2 版本化 + 一键恢复演练文档。

## 90 天落地路线图（建议）

- **第 1-2 周**：密码哈希迁移、JWT 密钥强校验、中间件验签。
- **第 3-4 周**：登录限流 + 审计日志表 + 高风险操作日志。
- **第 5-8 周**：RBAC 权限矩阵、API Key 配额/过期、告警面板。
- **第 9-12 周**：发布工作流、备份恢复演练、搜索体验增强。
